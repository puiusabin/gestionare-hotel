3Descriere Aplicatie
4
5Sistemul de rezervari hotel este o aplicatie web care permite utilizatorilor sa vizualizeze
6camere disponibile, sa faca rezervari si sa le gestioneze. Administratorii pot gestiona
7camerele disponibile in sistem.
8
9Obiective Principale:
10 - Facilitarea procesului de rezervare pentru clienti
11 - Gestionarea eficienta a camerelor de hotel
12 - Prevenirea suprapunerilor de rezervari
13 - Notificari automate prin email
14
15Arhitectura Aplicatiei
16
17Roluri Utilizatori:
18
19Guest (Vizitator/Client):
20 - Inregistrare cont nou
21 - Autentificare in sistem
22 - Vizualizare camere disponibile
23 - Cautare camere cu filtre
24 - Realizare rezervari
25 - Vizualizare propriile rezervari
26 - Anulare rezervari
27
28Admin (Administrator):
29 - Autentificare in sistem
30 - Adaugare camere noi
31 - Editare detalii camere
32 - Stergere camere
33 - Vizualizare toate rezervarile
34
35Entitati Principale:
36 - Users: utilizatori ai sistemului (clienti si administratori)
37 - Rooms: camere disponibile pentru rezervare
38 - Reservations: rezervari efectuate de clienti
39
40Componente Principale:
41 - Modul Autentificare: login/logout, sesiuni, control acces
42 - Modul Rezervari: verificare disponibilitate, creare rezervari
43 - Modul Gestiune Camere: adaugare/editare/stergere camere
44
45Procesul de Rezervare:
46 1. Clientul selecteaza datele de check-in si check-out
47 2. Sistemul cauta camerele disponibile
48 3. Clientul alege o camera si confirma rezervarea
49 4. Sistemul verifica din nou disponibilitatea
50 5. Rezervarea este creata in baza de date
51 6. Email de confirmare este trimis clientului
52
53Baza de Date
54
55Schema Bazei de Date:
56 - users → reservations (one-to-many)
57 - rooms → reservations (one-to-many)
58
59Tabel users:
60 - id (INT, PRIMARY KEY, AUTO_INCREMENT)
61 - email (VARCHAR(255), UNIQUE, NOT NULL)
62 - password (VARCHAR(255), NOT NULL, hashed)
63 - first_name (VARCHAR(100), NOT NULL)
64 - last_name (VARCHAR(100), NOT NULL)
65 - phone (VARCHAR(20))
66 - role (ENUM('guest', 'admin'), DEFAULT 'guest')
67 - created_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
68
69Tabel rooms:
70 - id (INT, PRIMARY KEY, AUTO_INCREMENT)
71 - room_number (VARCHAR(10), UNIQUE, NOT NULL)
72 - room_type (ENUM('single', 'double', 'suite'), NOT NULL)
73 - capacity (INT, NOT NULL)
74 - price_per_night (DECIMAL(10,2), NOT NULL)
75 - description (TEXT)
76 - image_url (VARCHAR(255))
77 - is_available (BOOLEAN, DEFAULT TRUE)
78
79Tabel reservations:
80 - id (INT, PRIMARY KEY, AUTO_INCREMENT)
81 - user_id (INT, FOREIGN KEY → users(id))
82 - room_id (INT, FOREIGN KEY → rooms(id))
83 - check_in_date (DATE, NOT NULL)
84 - check_out_date (DATE, NOT NULL)
85 - total_price (DECIMAL(10,2), NOT NULL)
86 - status (ENUM('pending', 'confirmed', 'cancelled', 'completed'))
87 - created_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
